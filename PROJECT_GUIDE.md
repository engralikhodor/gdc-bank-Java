## 1. System Overview

A cloud-native banking system built with Spring Boot 3 and React, featuring AI-driven financial insights and
automated cloud-based document management. The system architecture is designed for high availability and stateless
security.

---

## 2. Technical Stack

`Backend:` Java 17, Spring Boot 3, Spring Security, JPA/Hibernate.

`Frontend:` React (TypeScript), Vite, Server-Sent Events (SSE).

`Database:` PostgreSQL (Production) / H2 (Dev).

`Cloud: `Amazon S3 (Document Storage).

`Intelligence:` OpenAI API (GPT-3.5/4) with Resilience4j circuit breakers.

---

## 3. Core Architectural Modules

`A. Identity & Security (/auth)`

- Stateless Auth: Uses JWT for request authorization.

- Token Rotation: Provides a short-lived accessToken (15m) and a long-lived refreshToken (24h) stored in the database to
  manage persistent sessions securely.

- Nested Registration: A single POST /register creates both the Auth credentials and the Customer profile in a single
  atomic transaction.

`B. Banking Operations (/customers, /transactions)`

- Transactional Integrity: All transfers use @Transactional to ensure that if a "Debit" succeeds but a "Credit" fails,
  the
  entire operation rolls back (ACID compliance).

- Dynamic Search: Uses JPA Criteria API (Specifications). Instead of writing 20 different query methods, the search
  endpoints handle dynamic filtering (by email, age, or account number) through a single interface.

`C. Automated Billing & Cloud Storage (/bills)`

- Spring Scheduling: A background worker (BillSchedulerService) processes pending bills at midnight.

- Object Storage: Instead of bloating the database with text files, the system generates receipts and streams them to
  Amazon S3. Only the resulting URL is stored in the database.

`D. AI Intelligence (/ai)`

- Real-time Streaming: Uses Server-Sent Events (SSE). The UI doesn't wait for the whole analysis; it streams words as
  they
  are generated by OpenAI.

- Fault Tolerance: Integrated `Resilience4j` Circuit Breaker. If OpenAI is slow or down, the system triggers a fallback
  method to inform the user gracefully rather than timing out.

---

## 4. The Journey of a Request (Data Flow)

`Frontend:` App.tsx sends a fetch request with a Bearer token.

`Security Filter:` JwtAuthenticationFilter intercepts the request, validates the token, and sets the Security Context.

`Controller:` Validates the input using @Valid and passes DTOs to the Service.

`Service Layer:` Executes business logic (e.g., checking balance, calling AI fallbacks).

`Data Layer:` MapStruct converts DTOs to Entities, and JpaRepository persists data to the database.

---

## 5. Critical Environment Variables

To run this system, the following must be configured in application.yml or Environment Variables:

- JWT_SECRET: Minimum 256-bit key for HS256.

- OPENAI_API_KEY: For financial insights.

- AWS_ACCESS_KEY / AWS_SECRET_KEY: For S3 bucket access.

- AWS_BUCKET_NAME: The destination for bill receipts.
